### Coding Style Guidelines:-  
- Follow **C++17** standards and consistent naming:-
  - Classes → PascalCase (e.g., `HospitalManager`)
  - Variables & functions → camelCase
  - Constants → UPPER_CASE
  
1. Use 4 spaces for indentation (no tabs).
2. Curly braces on the same line as control statements.
3. Always include meaningful comments and Doxygen-style function headers.
4. Separate interface (`.h`) and implementation (`.cpp`) files.
5. Use consistent naming for Qt signals/slots (`on_<widget>_<event>`).
6. Ensure code passes `clang-format` checks before submission.
